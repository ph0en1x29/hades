🎯 The Attack (SIEM Pipeline Injection)

The core insight: SIEMs are faithful loggers. They don't sanitize — they record.

🔴 Attacker crafts HTTP header / DNS query / TLS cert with prompt injection → 📡 Network traffic hits target → 📋 SIEM logs it verbatim → 🔧 Normalizer includes payload in alert → 🤖 LLM triage reads poisoned alert → wrong decision
DimensionCountExamples
Injection Vectors12HTTP User-Agent, DNS TXT, TLS CN, hostname, URL path...
Attack Classes5Misclassification, confidence manipulation, reasoning corruption, attention hijacking, escalation suppression
Encodings9Plaintext, Base64, homoglyph, zero-width, leetspeak, ROT13...

🔒 Behavioral Invariants

Model-agnostic constraints that triage outputs must satisfy regardless of input manipulation.

IDCheckWhat It Catches
INV-1Classification in valid setInjection producing garbage labels
INV-2Confidence ∈ [0, 1]Confidence manipulation out of bounds
INV-3High confidence benign → flagSuppression attacks (hiding real threats)
INV-4MITRE technique consistencyTechnique injection / hallucination
INV-5Reasoning ≥ minimum lengthReasoning truncation / corruption
INV-6Confidence-severity alignmentC2 attacks INV-3 misses (low confidence on real threats)
