# Abstract

Large Language Models are increasingly deployed for automated alert triage in Security Operations Centers, processing thousands of SIEM alerts that overwhelm human analysts. This collapses a trust boundary that classic security pipelines maintain: attacker-controlled telemetry becomes model-interpretable, instruction-bearing context. We systematize this vulnerability, first demonstrated by Neaves [2025]: adversaries embed prompt injection payloads in network traffic fields — HTTP headers, DNS queries, TLS certificates — that SIEM systems feed directly to triage models. Attack traffic thus serves simultaneously as detection evidence and adversarial control input.

We present **Hades**, an open-source evaluation framework and benchmark for studying this threat. Hades constructs a benchmark of 12,147 rule-linked alerts across 27 MITRE ATT&CK techniques and defines an evaluation protocol for 4 open-weight models (3 MoE + 1 dense control), generating 1.4 million adversarial variants across 12 injection vectors and 5 attack classes under three attacker knowledge levels. Cross-model attack success evaluation awaits GPU allocation; this paper reports framework design, benchmark construction, SIEM survival validation, and preliminary defense results.

We propose two defense components: (1) a **correlator agent** for multi-stage campaign detection through IP clustering and technique chain matching; and (2) **output consistency checks** (behavioral invariants) operating on triage *outputs* rather than inputs. All 15 tested payload templates survive SIEM normalization across 11 rules without semantic loss; keyword defenses catch only 2 of 5 attack classes. In a preliminary study on 50 simulated triage outputs (not live model generations), invariants flagged 100% of direct misclassification (C1) and reasoning corruption (C3), 98% of attention hijacking (C4), with 0/50 false positives — though these wide confidence intervals and narrow technique coverage (single T1003.001 family) limit generalization. A counterfactual what-if analysis over manually perturbed outputs suggests injection could degrade Fox campaign scores by 53.7 points (95.7→42.0) assuming successful injection and no output-side defenses — this is not an observed end-to-end attack success rate. We release Hades as open-source tooling for evaluating LLM-based security automation.
