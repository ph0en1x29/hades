# Abstract

Large Language Models are increasingly deployed for automated alert triage in Security Operations Centers, processing thousands of SIEM alerts that overwhelm human analysts. This collapses a trust boundary that classic security pipelines maintain: attacker-controlled telemetry becomes model-interpretable, instruction-bearing context. We systematize this vulnerability, first demonstrated by Neaves [2025]: adversaries embed prompt injection payloads in network traffic fields — HTTP headers, DNS queries, TLS certificates — that SIEM systems feed directly to triage models. Attack traffic thus serves simultaneously as detection evidence and adversarial control input.

We present **Hades**, an evaluation framework targeting 4 open-weight models (3 MoE + 1 dense control) with 12,147 rule-linked alerts across 27 MITRE ATT&CK techniques, generating 1.4 million adversarial variants across 12 injection vectors and 5 attack classes under three attacker knowledge levels.

Hades introduces two defense components: (1) a **correlator agent** for multi-stage campaign detection through IP clustering and technique chain matching; and (2) **behavioral invariant checks** operating on triage *outputs* rather than inputs — detecting phantom IPs, fabricated references, and suspicious confidence patterns, auto-escalating suspected injections. All payloads survive SIEM normalization; keyword defenses catch only 2 of 5 attack classes. Preliminary evaluation on 50 simulated outputs shows invariants detecting 100% of direct misclassification (C1) and reasoning corruption (C3), 98% on attention hijacking (C4), with 0% false positives. A counterfactual impact analysis suggests injection could degrade Fox campaign scores by 53.7 points with defenses disabled (95.7→42.0), though this assumes successful injection rather than measuring observed model failure. Full cross-model evaluation awaits GPU allocation. We release Hades as open-source tooling for evaluating LLM-based security automation.
