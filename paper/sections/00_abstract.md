# Abstract

Large Language Models are increasingly deployed for automated alert triage in Security Operations Centers, processing thousands of SIEM alerts that overwhelm human analysts. This collapses a trust boundary that classic security pipelines keep separate: attacker-controlled telemetry becomes model-interpretable, instruction-bearing context. We systematize and evaluate this vulnerability, first demonstrated by Neaves [2025]: adversaries can embed prompt injection payloads in network traffic fields — HTTP headers, DNS queries, TLS certificates — that SIEM systems log and feed directly to triage models.

We present **Hades**, an evaluation framework for measuring and defending against adversarial manipulation of LLM-based SOC systems. Using 12,147 rule-linked alerts across 27 MITRE ATT&CK techniques, we generate 1.4 million adversarial variants across 12 injection vectors and 5 attack classes. Our framework targets 4 open-weight models (DeepSeek R1, GLM-5, Kimi K2.5, Qwen 3.5) — three MoE architectures plus one dense control — under three attacker knowledge levels.

Hades introduces two novel defense components: (1) a **correlator agent** detecting multi-stage campaigns through IP clustering and technique chain matching; and (2) **behavioral invariant defenses** that detect phantom IPs, fabricated references, and suspicious confidence patterns in triage outputs, auto-escalating suspected injections without relying on input-level defenses that adaptive attackers consistently bypass (Nasr et al., 2025). The framework also includes a **SOC-Bench adapter** for standardized ring-scored evaluation. Crucially, Hades demonstrates that attack traffic can serve simultaneously as detection evidence *and* as adversarial control input to the triage model.

Normalization analysis shows direct misclassification and confidence manipulation payloads survive 100% of SIEM normalization, while evasion encodings defeat keyword defenses yet remain LLM-interpretable. Preliminary evaluation on 50 simulated triage outputs from T1003.001 alerts shows behavioral invariants detecting 100% of direct misclassification (C1) and reasoning corruption (C3), 98% on attention hijacking (C4), with 0% false positives; robustness against adaptive attackers and real model outputs awaits GPU experiments. Pipeline dry-run validation shows adversarial injection can degrade Fox campaign scores by 53.7 points under simulated worst-case conditions with defenses disabled (95.7→42.0/100).

Corroborating evidence: Neaves (2025) demonstrated successful injection through HTTP User-Agent and SSH usernames in production SIEMs; Unit 42 (2026) reports 22 indirect prompt injection techniques in the wild. We release Hades as open-source tooling for evaluating LLM-based security automation.
